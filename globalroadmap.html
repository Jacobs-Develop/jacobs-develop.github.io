Global roadmap

This is simple explanation of how is looking my mod creation progress:


    CONCEPTS


    It's first and most important state in my mod development. This is state when I'm sketching concepts such as
    blocks and their functions, sprites and other things required to start with mod. Sometimes mod concept can be abandoed,
    but not everytime all sprites, blocks or functions are scrapped, they can be used in other mods or addons with changed functions or sprites.


    IN DEVELOPMENT


    It's second stage after "concepts" when mod is already on github and available to download.
    Even tho it's available to download this doesn't mean that mod cannot change, there can be added blocks, fluids, items
    and other content or some content can be changed. Mod in this state can sometimes use placeholders as sprites, that will
    be changed in future.


    RELEASED


    This is final stage after "in development" state. Mod is fully released and it's content
    is blocked which means that content (blocks, fluids, functions, etc.) will not be added anymore. Mostly content will be
    updated rerely new content will be added useless there is planned a big update that will add a big amount of content
    for modification.


    ABANDOED/SCRAPPED


    This is not a stage but still a one of possible mod states. This state is easy in explanation, mod is discontuined or
    just scrapped which means that mod will never be released, only content from scrapped mods can be used in future mods
    or in already functional mods. Mostly this state can be set after "concepts" state or even in
    "in development" state, but this is less probable to happen.
